# Production Deployment — memoriahub.marin.cr

This guide covers deploying MemoriaHub to production on a VPS using the scripts in
`infra/deploy/`. For the development deployment (`memoriahub.dev.marin.cr`, shared
devnet Postgres, hot-reload Vite), see [docs/ssl-nginx-setup.md](ssl-nginx-setup.md).

---

## Architecture

### Two-tier reverse proxy

```
Internet  HTTPS :443
    |
    v
VPS host-level nginx (proxy container: proxy-nginx)
    /opt/infra/proxy/nginx/conf.d/memoriahub.conf
    SSL certs: /opt/infra/proxy/letsencrypt/live/memoriahub.marin.cr/
    |
    | 127.0.0.1:8328   (host-local only)
    v
Docker Compose (memoriahub-internal network)
    +-- nginx  (nginx:alpine, nginx.prod.conf, :80)
    |       /api  ->  api:3000
    |       /     ->  web:80
    +-- api   (NestJS + Fastify, production build target, 512 MB limit)
    +-- web   (React static files served by nginx :80, 128 MB limit)
```

### Key differences from dev

| Aspect | Development | Production |
|---|---|---|
| Domain | `memoriahub.dev.marin.cr` | `memoriahub.marin.cr` |
| Host proxy | systemd nginx + wildcard `*.dev.marin.cr` map | containerized `proxy-nginx` + per-domain config |
| SSL cert | wildcard `*.dev.marin.cr` (Let's Encrypt, shared) | per-domain `memoriahub.marin.cr` (Let's Encrypt, managed by installer) |
| Internal port | 8327 | 8328 |
| Web frontend | Vite dev server `:5173` | static nginx `:80` (production build target) |
| Database | shared `devnet` Postgres container | external/cloud PostgreSQL over SSL |
| Compose files | `base.compose.yml` + `dev.compose.yml` | generated `compose.yml` (standalone) |
| Hot reload | yes | no |

### Internal nginx

Production uses `infra/nginx/nginx.prod.conf` (mounted read-only into the nginx
container). It routes `/api` to `api:3000` and `/` to `web:80`. `client_max_body_size`
is `10g` to match the host-level proxy and `MAX_FILE_SIZE` application setting.
Resumable multipart uploads use pre-signed S3 URLs and bypass nginx entirely.

### Database

Production connects to an external cloud-hosted PostgreSQL. The `memoriahub-internal`
Docker network has **no** Postgres container; the API connects outbound using the
`POSTGRES_*` variables with `POSTGRES_SSL=true`. `DATABASE_URL` is constructed at
runtime by `apps/api/scripts/prisma-env.js`.

### Storage

Media is stored in the `marin-memoriahub` S3 bucket. See
[`infra/aws/README.md`](../infra/aws/README.md) for bucket provisioning and CORS setup.

## Install directory layout

```
/opt/infra/apps/memoriahub/
  repo/                    # git clone of main branch
  .env                     # production environment variables (NOT in git)
  compose.yml              # generated by install.sh (do not edit manually)
  memoriahub.conf          # generated VPS proxy config (copy to proxy conf.d)
  logs/                    # update.sh timestamped logs (newest 10 kept)
  .update-state            # written after each update.sh run
```

## Prerequisites

Before running the installer, ensure:

1. **VPS proxy** — `/opt/infra/proxy/` exists with:
   - `nginx/conf.d/` directory (where `memoriahub.conf` will be placed)
   - `webroot/` directory (ACME challenge root)
   - `letsencrypt/` directory (cert storage)
   - A running `proxy-nginx` Docker container serving :80 and :443
2. **DNS** — `memoriahub.marin.cr` resolves to the VPS IP
3. **Cloud PostgreSQL** — accessible from the VPS; SSL enabled
4. **Google OAuth** — redirect URI registered:
   `https://memoriahub.marin.cr/api/auth/google/callback`
5. **S3 bucket** — `marin-memoriahub` provisioned with correct CORS and IAM credentials
6. **Docker + Docker Compose v2** — installed on the VPS

## Environment file

The installer expects `.env` at `${MEMORIAHUB_HOME}/.env` (default:
`/opt/infra/apps/memoriahub/.env`) before it will proceed past step 4.

Create it from the template in the cloned repo:

```bash
cp /opt/infra/apps/memoriahub/repo/infra/compose/.env.example \
   /opt/infra/apps/memoriahub/.env
nano /opt/infra/apps/memoriahub/.env
```

Required production values (see `.env.example` for all variables and defaults):

```bash
# Application
NODE_ENV=production
APP_URL=https://memoriahub.marin.cr

# Database — cloud PostgreSQL
POSTGRES_HOST=<your-cloud-pg-host>
POSTGRES_PORT=5432
POSTGRES_USER=<user>
POSTGRES_PASSWORD=<password>
POSTGRES_DB=<dbname>
POSTGRES_SSL=true

# JWT / Session — generate with: openssl rand -base64 32
JWT_SECRET=<min-32-chars>
COOKIE_SECRET=<min-32-chars>

# Google OAuth
GOOGLE_CLIENT_ID=<client-id>
GOOGLE_CLIENT_SECRET=<client-secret>
GOOGLE_CALLBACK_URL=https://memoriahub.marin.cr/api/auth/google/callback

# Initial admin
INITIAL_ADMIN_EMAIL=<your-email>

# Storage
STORAGE_PROVIDER=s3
S3_BUCKET=marin-memoriahub
S3_REGION=<region>
AWS_ACCESS_KEY_ID=<key-id>
AWS_SECRET_ACCESS_KEY=<secret-key>
MAX_FILE_SIZE=10737418240
ALLOWED_MIME_TYPES=image/*,video/*

# App configuration
GEO_PROVIDER=offline
DEVICE_PAT_TTL_DAYS=90
COMPOSE_PROJECT_NAME=memoriahub
```

## First install

### Option A — run directly from the repo

```bash
mkdir -p /opt/infra/apps/memoriahub
cd /opt/infra/apps/memoriahub

# Clone the repo manually so you can place .env before the installer runs
git clone https://github.com/marinoscar/MemoriaHub.git repo

# Place and configure .env (see Environment file section above)
cp repo/infra/compose/.env.example .env
nano .env

# Run the installer
bash repo/infra/deploy/install.sh
```

### Option B — one-liner (requires .env to exist first)

```bash
# First ensure .env exists at /opt/infra/apps/memoriahub/.env, then:
bash <(curl -fsSL https://raw.githubusercontent.com/marinoscar/MemoriaHub/main/infra/deploy/install.sh)
```

### What the installer does (9 steps)

| Step | Action |
|---|---|
| 1 | Check `docker`, `git`, `curl`, and `docker compose` v2 |
| 2 | Create `repo/` and `logs/` directories under `MEMORIAHUB_HOME` |
| 3 | Idempotent git clone or `reset --hard origin/main` |
| 4 | Validate `.env` exists; print required key list if missing |
| 5 | Generate `compose.yml` with production build targets, `127.0.0.1:8328:80`, resource limits, and `memoriahub-internal` network |
| 6 | Generate `memoriahub.conf` — VPS proxy config with SSL cert paths, security headers, `/api` + `/` proxy rules, HTTP→HTTPS redirect, and ACME challenge block |
| 7 | `docker compose build` (api and web production targets) |
| 8 | `docker compose run --rm api npm run prisma:migrate` then `npm run prisma:seed` |
| 9 | `docker compose up -d` then health check (`wget` to `localhost:3000/api/health/live`, up to 120 s) |

### Post-install checklist

The installer prints this checklist after a successful run. Complete each step:

**1. Copy the VPS proxy config**

```bash
cp /opt/infra/apps/memoriahub/memoriahub.conf \
   /opt/infra/proxy/nginx/conf.d/memoriahub.conf
```

**2. Issue the TLS certificate**

```bash
certbot certonly --webroot \
  -w /opt/infra/proxy/webroot \
  -d memoriahub.marin.cr \
  --config-dir /opt/infra/proxy/letsencrypt
```

**3. Test and reload the VPS proxy**

```bash
docker exec proxy-nginx nginx -t
docker exec proxy-nginx nginx -s reload
```

**4. Register the Google OAuth redirect URI**

In [Google Cloud Console](https://console.cloud.google.com/apis/credentials), add:
```
https://memoriahub.marin.cr/api/auth/google/callback
```

**5. Confirm DNS**

```bash
dig +short memoriahub.marin.cr   # should return the VPS IP
```

**6. Verify end-to-end**

```bash
curl https://memoriahub.marin.cr/api/health/live
# Expected: {"data":{"status":"ok"}}
```

Then open `https://memoriahub.marin.cr/` and log in with the `INITIAL_ADMIN_EMAIL`
account, which becomes Admin on first login. Swagger UI is at
`https://memoriahub.marin.cr/api/docs`.

## Updates

```bash
cd /opt/infra/apps/memoriahub
bash repo/infra/deploy/update.sh
```

The updater:

1. Fetches `origin/main` and compares commits — **exits early** if already up to date
   (unless `--no-cache` is passed)
2. Rebuilds Docker images
3. Stops the API, runs `prisma migrate deploy` (additive only, never resets), restarts all services
4. Restarts nginx to re-resolve upstream container IPs
5. Optionally copies `memoriahub.conf` to the proxy `conf.d` and reloads `proxy-nginx`
   if the file differs (skip with `--skip-proxy`)
6. Runs a health check and writes `.update-state`

All output is also written to a timestamped log in `logs/`; the newest 10 log files
are kept.

### Updater flags

| Flag | Effect |
|---|---|
| `--no-cache` | Force full Docker rebuild (bypasses layer cache); also skips the up-to-date early exit |
| `--skip-proxy` | Skip proxy config diff/copy/reload step |
| `--no-color` | Disable ANSI output (useful in CI or non-TTY contexts) |
| `--help` | Print help |

## Installer/updater environment variable overrides

Both scripts read these variables from the calling shell; none are required if defaults
are acceptable.

| Variable | Default | Purpose |
|---|---|---|
| `MEMORIAHUB_DOMAIN` | `memoriahub.marin.cr` | Domain used in generated nginx configs and checklist output |
| `MEMORIAHUB_PORT` | `8328` | Host port bound to `127.0.0.1` (`127.0.0.1:PORT:80` in compose.yml) |
| `MEMORIAHUB_REPO` | `https://github.com/marinoscar/MemoriaHub.git` | Git clone URL |
| `MEMORIAHUB_REF` | `main` | Branch or tag to deploy |
| `MEMORIAHUB_HOME` | `/opt/infra/apps/memoriahub` | Install root directory on the VPS |
| `GITHUB_TOKEN` | _(empty)_ | GitHub PAT for private-repo clones (install.sh only) |

Example — deploy a release tag to a custom directory:

```bash
MEMORIAHUB_REF=v1.2.0 \
MEMORIAHUB_HOME=/opt/apps/memoriahub \
bash repo/infra/deploy/install.sh
```

## Operational reference

### Container names

| Container | Image / build target | Role |
|---|---|---|
| `memoriahub-nginx` | `nginx:alpine` | Internal reverse proxy |
| `memoriahub-api` | `apps/api Dockerfile (production)` | NestJS API |
| `memoriahub-web` | `apps/web Dockerfile (production)` | React static files |

### Useful commands

```bash
# Tail API logs
docker compose -f /opt/infra/apps/memoriahub/compose.yml logs -f api

# Check container status
docker compose -f /opt/infra/apps/memoriahub/compose.yml ps

# Run a one-off API command
docker compose -f /opt/infra/apps/memoriahub/compose.yml run --rm api <command>

# Force-rebuild without updating code
NO_CACHE=true bash repo/infra/deploy/update.sh --no-cache

# Reload proxy after manual config edits
docker exec proxy-nginx nginx -t && docker exec proxy-nginx nginx -s reload
```

### Health endpoints

| Endpoint | Purpose |
|---|---|
| `GET /api/health/live` | Liveness — returns `{"data":{"status":"ok"}}` if the process is up |
| `GET /api/health/ready` | Readiness — includes a database connectivity check |

## Troubleshooting

**502 Bad Gateway from the VPS proxy**

Containers may not be running. Check:
```bash
docker compose -f /opt/infra/apps/memoriahub/compose.yml ps
docker compose -f /opt/infra/apps/memoriahub/compose.yml logs api
```

**API health check times out after install**

The API waits up to 120 s. If it fails, check for database connection errors:
```bash
docker compose -f /opt/infra/apps/memoriahub/compose.yml logs api | grep -i "error\|connect"
```
Confirm `POSTGRES_HOST`, `POSTGRES_SSL=true`, and that the cloud Postgres firewall
allows the VPS IP.

**Migrations fail**

The one-off migration container inherits the `.env`. If `prisma:migrate` fails,
run it manually to see the full output:
```bash
docker compose -f /opt/infra/apps/memoriahub/compose.yml run --rm api npm run prisma:migrate
```

**SSL certificate not yet issued**

The proxy will return a connection error until step 2 of the post-install checklist
(certbot) is complete. The ACME challenge block in `memoriahub.conf` handles
`/.well-known/acme-challenge/` over HTTP before the HTTPS server block is active.

**413 Request Entity Too Large**

Both the VPS proxy (`memoriahub.conf`) and the internal nginx (`nginx.prod.conf`) set
`client_max_body_size 10g`. Verify both are in place. For very large files, use the
resumable upload flow (`POST /api/storage/objects/upload/init`), which streams directly
to S3 and bypasses nginx entirely.

**Google OAuth errors**

Verify the exact redirect URI `https://memoriahub.marin.cr/api/auth/google/callback`
is listed in Google Cloud Console and matches `GOOGLE_CALLBACK_URL` in `.env`.
